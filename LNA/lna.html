Project Overview

This project showcases a structured Learning Needs Analysis template created to support instructional designers and L&D teams in identifying performance gaps, diagnosing root causes, and determining whether training is the appropriate solution.

Audience

Instructional Designers, L&D Consultants, Academic Designers, QA and Strategy Teams.

Deliverable

Multi-page Learning Needs Analysis template (interactive & print-ready PDF).

Tools Used

Adobe InDesign, UX writing principles, performance consulting frameworks.

My Role

Instructional Designer, Learning Experience Designer, UX Architect.

Instructional Design Methodology

The template is grounded in performance consulting and Learning Needs Analysis best practices. It intentionally separates problem framing, performance analysis, root cause diagnosis, and solution recommendations.

Rather than assuming training as the default solution, the template supports evidence-based decisions by prompting designers to consider non-training interventions such as job aids, process changes, and environmental constraints.

The overall flow aligns with ADDIE (Analysis phase) and Action Mapping principles, ensuring that any subsequent learning design is directly tied to observable performance outcomes.

Template Structure

  • Purpose & orientation (why the analysis is conducted)
  • Business context and stakeholder framing
  • Target audience and performance environment analysis
  • Current vs desired performance comparison
  • Root cause analysis and training decision logic
  • Recommended solutions and learning strategy
  • Constraints, risks, and opportunities
  • Analysis summary, recommendations, and next steps

Design Decisions & UX Rationale

Cognitive Load Management

Each page focuses on a single thinking task to reduce overload and prevent premature solution bias. White space is used intentionally to support reflection and clarity.

UX Writing & Prompts

Prompt-based questions guide analytical thinking while avoiding prescriptive language. Sentence case is used to maintain a conversational, non-intimidating tone.

Visual Hierarchy

A restrained typographic scale and consistent alignment system ensure scanability for both designers and stakeholders reviewing the analysis.

Real-world Usability

The template was designed to function equally well as a facilitated workshop document or a self-guided analysis tool in real projects.
