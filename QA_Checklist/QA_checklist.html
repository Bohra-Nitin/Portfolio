eLearning QA Checklist

A UX-driven, interactive Quality Assurance checklist designed in Adobe InDesign for reviewing eLearning courses. Built with cognitive load, accessibility, and real QA workflows in mind.

View Interactive PDF

Project Overview

This project showcases a comprehensive eLearning Quality Assurance Checklist created as an interactive PDF using Adobe InDesign. The checklist enables instructional designers, QA reviewers, and L&D teams to systematically evaluate instructional quality, usability, accessibility, media, performance, LMS tracking, and assessment integrity.

Audience

Instructional Designers, L&D QA reviewers, eLearning developers, academic QA teams.

Deliverable

Interactive PDF QA checklist (print-safe and digitally fillable).

Tools Used

Adobe InDesign, Interactive PDF forms, UX heuristics, accessibility guidelines.

My Role

Instructional Designer, UX Designer, QA Framework Architect.

Checklist Structure

  • Row-based checklist layout instead of rigid tables
  • Yes / No / N/A response model with mandatory remarks
  • Full-width row containers for visual grouping
  • Top-aligned controls for long checklist items
  • Interactive PDF fields compatible across viewers

QA Checklist Sections

  • Orientation: Cover & How to Use
  • Instructional Quality: Alignment & Content Accuracy
  • UI & Navigation: Visual design and controls
  • Accessibility: Inclusive and WCAG-aligned checks
  • Interactivity & Media: Feedback and media quality
  • Performance: Stability and load behavior
  • LMS Tracking: Completion, scoring, reporting
  • Assessment: Validity, feedback, scoring logic
  • Final Sign-off: QA approval and accountability

Design Rationale

This QA checklist was designed using UX and cognitive load principles to support accurate, repeatable reviews of eLearning courses.

A row-based layout was chosen over tables to improve scanability and accommodate variable-length checklist items without breaking alignment.

High-risk areas such as accessibility, assessment quality, and LMS tracking were given focused attention to reduce evaluation errors.

Interactive PDF fields were implemented to support real-world QA workflows while remaining print-friendly.
